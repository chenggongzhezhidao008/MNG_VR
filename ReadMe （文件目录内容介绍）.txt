各个文件封装功能介绍：

1.configmodule文件：
 主要实现 软件启动后对配置文件（以.ini为后缀名称的文件）的读取和解析功能。

2.datadue 文件：
  主要实现 软件在数据存取操作时 对数据的存取管理（增删改查数据）功能。

3.distanceTransfer 文件：
  主要实现 软件基于CDMA4G模块传输设备 在远程数据交互上的相关数据传输与控制功能。

4.image 文件
  主要实现 以CSS脚本文件进行配置软件在启动后的界面上的友好性及美化功能。

5.logtxt 文件
  主要实现 将采集到的数据进行分类及按照指定格式存储于Txt文本文件。

6.mess 文件
  主要实现 将第一手接入的数据按照附件9民航数据格式进行消息头解析的功能。

7.MessageParse 文件
  主要实现 整个数据（除头外的）体进行解析出用户数据。

8.serialmodule 文件
  主要实现 创建一个独立于主线程的子线程，采用多线程技术实现串口数据的收发及对串口的管理（通讯参数：波特率，端口...）.

9.seting 文件
  主要实现 以.ini配置脚本文件形式，给系统提供所需运行管理参数。


8.sqlite 文件
  主要实现 创建一个独立于主线程的子线程，采用多线程技术实现数据库数据的存储及管理（数据文件检查，创建及插入等）.

9.ui 文件
  主要实现 整个软件用户交互所需UI框架。

10.WindPlugins 文件
  主要实现 风数据展示界面控件的绘制及数据展示。

11.XLSX 文件
  主要实现 对EXCEL文件的读取及写入过程中相关的功能。

12 ZMQ 文件
  主要实现 以ZMQ为开发库的TCP网络数据交互功能。
